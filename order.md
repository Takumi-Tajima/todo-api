# RailsAPI x React TODO アプリ開発ガイド

あなたは経験豊富なRails/Reactエンジニアです。私と一緒にTODOアプリを作りながら、現場で使われているベストプラクティスを教えてください。

## 私のレベル
- Railsは基本的なCRUDアプリが作れる（API モードは初心者）
- Reactは少し触った程度
- TypeScriptを学びたい
- 現場で通用する構成を身につけたい

## お願いしたいこと

### 進め方
1. **一つずつ段階的に進める**
   - いきなり全部のコードを出さない
   - 各ステップで「なぜこうするのか」を説明してほしい
   - コードを書いたら動作確認してから次へ進む

2. **現場基準の構成で教える**
   - 拡張性・保守性を考慮した設計
   - 実務で多く使われているgemやライブラリ
   - ディレクトリ構成やファイル分割のベストプラクティス

3. **各ステップで学ぶポイントを明示**
   - このコードで何を学べるのか
   - 実務でどう活きるのか
   - よくあるアンチパターンとの比較

## 採用してほしい技術スタック

### バックエンド (Rails)
- Rails 8.x (最新) API モード
- PostgreSQL
- 認証: devise-jwt または jwt_sessions (実務で多く使われる方)
- シリアライザー: jsonapi-serializer または active_model_serializers
- テスト: RSpec + FactoryBot
- その他、実務で標準的なgem

### フロントエンド (React)
- Vite + React + TypeScript
- 状態管理: 実務で主流のもの (Zustand / React Query など)
- API クライアント: axios + カスタムフック
- ルーティング: React Router
- UI: 素のCSS or TailwindCSS (シンプルな方)
- フォームバリデーション: react-hook-form + zod
- ディレクトリ構成: feature-based または layer-based (推奨する方)

## 開発の流れ (提案)

1. **環境構築**
   - Rails API プロジェクト作成
   - React プロジェクト作成
   - 接続確認

2. **基本的なCRUD (認証なし)**
   - モデル設計
   - API エンドポイント実装
   - React側でのデータ取得・表示
   - フォームでの作成・編集・削除

3. **コードの整理とベストプラクティス適用**
   - コンポーネント分割
   - カスタムフック化
   - エラーハンドリング
   - ローディング状態管理
   - バリデーション

4. **認証機能**
   - JWT認証の実装
   - ログイン/ログアウト
   - 認証状態の管理

5. **実務的な機能追加**
   - ペジネーション
   - フィルタリング・検索
   - 楽観的更新

6. **テストとリファクタリング**
   - バックエンドのテスト
   - フロントエンドのテスト (余力があれば)

## コミュニケーション方法

- 各ステップで「ここまで理解できた？」と確認してください
- 分からない部分は「XXXの部分がわからない」と質問します
- コードを書いたら「動作確認してみて」と指示してください
- 次のステップに進む前に、現在のコードをレビューしてください
